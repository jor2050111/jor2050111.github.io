CIS133 · The radio group
No wrapper declares the group

The group is the name.

<input type="radio" id="type-drop-in"
       name="session-type" value="drop-in">
<label for="type-drop-in">Drop-in</label>

<input type="radio" id="type-appointment"
       name="session-type" value="appointment">
<label for="type-appointment">Appointment</label>

Rename one and the group splits: both circles select, and picking releases nothing.

CIS133 · Naming the question
The group's own label contract

Labels name options. The legend names the question.

<fieldset>
  <legend>Which days can you meet?</legend>
  <p>
    <input type="checkbox" id="day-monday"
           name="meeting-day" value="monday">
    <label for="day-monday">Monday</label>
  </p>
</fieldset>

Devon hears "Which days can you meet? Monday", never a bare "Monday".

CIS133 · GET and POST
A judgment call you now own

Where the pairs ride decides which method fits.

GETpairs ride the URL · safe to repeat, bookmark, share · a search, a filter, a lookup

POSTpairs ride inside the request · private or state-changing · a signup, a password, an order

Either way, Chapter 1's law holds: never collect anything over plain http://.

CIS133 · The browser says no
Claims, checked before the trip

Two refusals, and the trip never started.

<label for="student-email">Email (required)</label>
<input type="email" id="student-email"
       name="student-email" required autocomplete="email">

The browser checks the claim's shape. It can never check ownership. That job waits for a server.

CIS133 · The vanishing hint
A hint, never a label

The label survives every state the placeholder cannot.

  • Placeholder: lives inside the box, gone at the first keystroke
  • Label: its own element, standing through every state
  • Required: marked in label text, never in color alone
CIS133 · Dressing the form
Owned selectors only

One column, restated type, and a ring you can measure.

label { display: block; margin: 16px 0 4px; }

input, select, textarea {
  width: 100%;
  padding: 8px;
  font-size: 1rem;   /* controls ignore the body until told */
}

input:focus, select:focus, textarea:focus {
  outline: 2px solid #268080;  /* 3.04:1, measured */
}
CIS133 · Retrieval check
Amend the proposal

"input:focus { outline: none; } · the default ring clashes with the brand."

Fix it: amend the rule so the brand improves and the law holds. Who does your amendment protect?

Replace, never remove: outline: 2px solid #268080; restyles the signal in the palette · it protects the keyboard visitor, who steers by that ring
