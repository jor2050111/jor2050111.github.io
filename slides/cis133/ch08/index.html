CIS133 · At least, at most
Say it aloud

Two features, mirror images.

@media (max-width: 600px) “At most 600 wide”

Fires from 0 up to the 600 tick, then sleeps.

@media (min-width: 600px) “At least 600 wide”

Fires from the 600 tick up, without limit.

020040060080010001200

Each feature includes the width it names.

CIS133 · Retrieval check
The boundary

The window is exactly 600 pixels wide.

Which fires: (max-width: 600px), (min-width: 600px), or both?

Both. 600 is at most 600, and 600 is at least 600. Each feature includes the width it names.

CIS133 · Base plus overrides
Only the difference

An override never repeats the base.

/* Base: every width the queries leave alone */
.update-card {
  padding: 20px;
}
/* Override: only the difference */
@media (max-width: 600px) {
  .update-card {
    padding: 10px;
  }
}

Copy all five declarations into the block, and you plant next month's disagreement.

CIS133 · Source order law
Why the override wins

Queries do not outrank. They come later.

1Base rulesserve every screen

✕Query block above the basethe base gets the last word, overrides never show
