CIS133 · 1.1
Two layers, one experience

The Internet carries the message. The web organizes the pages.

Networks, cables, servers, and devices make the Internet. HTML, HTTP, browsers, pages, and links make the World Wide Web.

CIS133 · 1.1
The client-server model

Clients ask. Servers answer.

Client

Your device makes the request.

A laptop, phone, TV, or other connected device can act as a client when it asks for a page.

Server

A connected computer sends the files.

A web server stores site files and returns the page a browser requested.

Quick check: When you open a site on your phone, which side is the client?

CIS133 · 1.1
The round trip

A page request follows a path.

1. Start

Enter an address or select a link.

2. Locate

DNS finds the server for the address.

3. Request

The browser asks the server for a page.

4. Return

The server sends the requested files.

5. Render

The browser turns code into a page.

CIS133 · 1.1
What your browser does

Request, receive, render.

A browser asks the right server for a page, receives the files that return, then renders HTML and CSS into the page you see.

Try it: View the source of any page. What did the server send that the browser turned into the finished page?

CIS133 · 1.2
Web addresses

A URL gives the browser a complete destination.

https://wwwphoenixcollege.edu/degrees-certificates

Protocol

Rules for how the browser and server communicate.

Domain

A human-readable name for a site or organization.

Path

The specific page or file the browser should request.

CIS133 · 1.2
The translation service

DNS turns a name you can read into an address computers can route.

You request phoenixcollege.edu. DNS returns an IP address. Then your browser knows where to send the page request.

Two failures can look different.

A DNS failure means the browser cannot find an address. An HTTP 404 means it reached a server, but the requested page is missing.

CIS133 · 1.2
Address book for the Internet

Names guide people. IP addresses guide packets.

DNS keeps the web usable because you can remember a domain name while network devices use an IP address to reach the right server.

CIS133 · 1.3
Finding information

Search engines do more than match words.

01

Crawl

Automated programs follow links and read pages.

02

Index

The engine stores what it found in a searchable catalog.

03

Rank

It orders matching results by predicted usefulness.

Remember: A high rank does not prove a source is accurate.

Find better results, then test what you found.

CIS133 · 1.3
Search results are starting points

Trust comes from checking, not clicking.

Ask who wrote the page, who hosts it, when it changed, what it wants from you, and what evidence supports its claims.

CIS133 · 1.4
Build and browse responsibly

Every web choice affects someone.

Copyright

Credit creators and learn the limits of fair use.

HTTPS

Look for encrypted connections and valid certificates.

Privacy

Notice what data a site collects and why.

Accessibility

Build pages people can use with different devices and needs.

Developer mindset: Clear structure, useful text, and secure choices help every visitor.
