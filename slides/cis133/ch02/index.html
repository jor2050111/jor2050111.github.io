CIS133 · 2.2
The required skeleton

Every page starts with the same nested structure.

The head informs the browser. The body holds what visitors see. The comment speaks only to humans.

<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="UTF-8">
        <title>My Study Log</title>
    </head>
    <body>
        <!-- CIS133 Chapter 2 practice page -->
        <h1>My Study Log</h1>
        <p>I am building web skills.</p>
    </body>
</html>
CIS133 · 2.2
Nesting makes the structure visible

First opened means last closed.

An element opened inside another element must close before its parent closes.

<p>Bring <strong>working chargers</strong>.</p>
CIS133 · 2.3
Choose by meaning

Structure answers what the content is, not how it looks.

Heading

Build an honest outline.

Use one <h1>. Do not skip heading levels.

Paragraph

Wrap running text.

Blank lines collapse. Two paragraphs need two <p> elements.

List

Apply the shuffle test.

If order matters, use <ol>. Otherwise, use <ul>.

CIS133 · 2.3
The shuffle test, run live

Both lists validate. Only one protects the reader.

the steps as <ol>
  1. Back up your files.
  2. Sign out, then factory reset.
  3. Remove any SIM or memory card.

Numbers promise the sequence holds.

same items as <ul>, shuffled
  • Sign out, then factory reset.
  • Back up your files.
  • Remove any SIM or memory card.

Reset before backup erases the files.

The validator passes both. The meaning choice is yours.

CIS133 · Retrieval check
Diagnose before you edit

A recipe uses a bulleted list and passes validation. Is the markup finished?

Discuss: What did the validator prove, and what decision did it never test?

The validator proved that the markup follows HTML grammar. It did not judge meaning. Recipe steps break when shuffled, so an ordered list is the stronger choice.
