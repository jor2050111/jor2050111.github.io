Cells and the kernel

The kernel remembers what you run.

[1]:x = 10
[2]:print(x + 5)
15
→

Kernel memory

x = 10 pandas as pd

Every run updates that memory. Every later cell can read it. Restarting the kernel clears it all.

CIS215 · Predict first
Predict first

Page order or run order?

[2]:print(x + y)
Ran second
[1]:x = 10 y = 20
Ran first

Commit: The top cell sits first on the page but ran second. Does it print 30, or does it fail?

It prints 30. The kernel follows run order, not page position. Restart the kernel and run the top cell first, and you get NameError: name 'x' is not defined.
CIS215 · 2.2
Two modes

Esc selects the cell. Enter goes inside.

Command mode

[ ]:df_sales.head()

Blue or grey border, no cursor. Keys act on the cell: add, delete, convert, move.

Esc brings you here.

Edit mode

[ ]:df_sales.head()

Green border, cursor inside. Keys type characters into the cell.

Enter brings you here.

CIS215 · 2.2
Command-mode keys

Five keys carry the workflow.

B

Below

Add a new cell below the current one.

DD

Delete

Remove the cell. Press D twice quickly.

M

Markdown

Turn the cell into formatted text.

Y

Code

Turn the cell back into Python.

ShiftEnter

Run

Run the cell and select the next one.

Practice until your hands know them.

CIS215 · 2.2
Tab completion

Type the dot. Press Tab. Choose.

  • After the dot, Tab lists every method the object offers.
  • Type a prefix first: .so narrows the list to sort_values().
  • Name DataFrames df_ and Tab after df_ lists them all.
CIS215 · 2.2
Tooltips

Shift+Tab reveals the signature.

sort_values(by, *, axis=0, ascending=True, inplace=False, ...)
Required. You supply it. Optional. The default is shown.

Place the cursor on the method name, press Shift+Tab, and read what it needs before you guess.

CIS215 · 2.3
Errors

An error is information, not a verdict.

Syntax error

The code never starts.

The grammar is broken: a missing bracket, quote, or colon. Python cannot tell what you are asking.

Runtime error

The code fails midway.

The grammar is fine, but a step cannot finish: a variable or column that does not exist.

Skilled analysts read error messages for a living. The message tells you where to look.

CIS215 · 2.3
Read the traceback

Three answers hide in every error.

KeyError                 Traceback (most recent call last)
Cell In[3], line 8
----> 8 df_cities['populaton']

KeyError: 'populaton'

What kind?

KeyError. A lookup by name failed.

Where?

line 8. The arrow points at the code that failed.

What exactly?

'populaton'. Read it letter by letter.

CIS215 · 2.3
The usual suspects

Four names cover most days.

SyntaxError

print("Hello"

A bracket or quote never closed.

NameError

print(total)

That variable does not exist yet.

KeyError

df['populaton']

No column or key by that name.

TypeError

'5' + 10

Those two types do not mix.

CIS215 · 2.3
The debugging loop

Fix one error at a time.

Read

The message names the problem. Start at the last line.

Identify

Which error type is it? That sets the strategy.

Locate

Follow the line number and the arrow.

Fix

Change one thing. Resist rewriting everything.

Rerun

Shift+Enter. Repeat until the cell runs clean.

One error at a time is how everyone works, including professionals.

CIS215 · Predict the error
Predict the error

Name it before you run it.

people_data = {'name': ['Amara', 'Kai']
df_people = pd.DataFrame(people_data)

Commit: One character is missing. Which error type appears, and where will Python point?

SyntaxError. The dictionary’s closing brace never arrives. Python reports that the { was never closed. The fix: end line 1 with ]}.
