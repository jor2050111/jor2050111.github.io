CIS215 · 3.1
Tuples

Lock the values that must hold still.

red_color = (255, 0, 0)
red_color[0] = 200

TypeError: 'tuple' object does not
support item assignment

Reach for a tuple when

  • GPS coordinates must stay fixed.
  • An RGB color is a set trio.
  • A record ID must never drift.

Reading works like a list. Writing raises the error, on purpose.

A test that is true or false, ending with a colon.

CIS215 · 3.3
Two loop partners

Count with range. Unpack with items.

range(start, stop, step)

range(5) → 0 1 2 3 4 range(1, 6) → 1 2 3 4 5 range(0, 11, 2) → 0 2 4 6 8 10

The stop is excluded. Slicing taught you this rule already.

.items()

for key, value in patient.items():
    print(f"{key}: {value}")

The loop receives each key and its value together.

CIS215 · 3.3
while loops

Repeat until the condition fails.

while temperature > 100.4:
    print("cooling...")
    temperature -= 0.5

# stops at 100.0°F
Warning

The exit must be reachable.

The body has to push the condition toward false. Drop the temperature -= 0.5 line and the loop runs forever.

for fits a known collection. while fits an unknown count.

CIS215 · 3.3
Loop control

Exit early or skip one.

break ends the loop

if temp > 100.4:
    print("Critical found")
    break

Stop scanning the moment you find what you came for.

continue skips one item

if temp > 100.4:
    continue
print(f"Valid: {temp}")

Pass over this item and keep the loop moving.

List comprehensions

The loop, folded into a line.

The loop you know

high_temps = []
for temp in temperatures:
    if temp > 100.0:
        high_temps.append(temp)

The same work, one line

high_temps = [temp
    for temp in temperatures
    if temp > 100.0]
expression for item in collection if condition

When the logic stops fitting on one readable line, return to the loop.
